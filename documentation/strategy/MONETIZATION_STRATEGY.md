# PixieTrack - Monetization Strategy

## Current Policy

- PixieTrack is fully free to use.
- Donations are always allowed and always optional.
- Core experience remains accessible without payment.

## Donation Channels

- **GitHub Sponsors** (active in-app card + profile link)
- **UPI QR Code** (Phase 1, India-first)
- Ko-fi (backup)

See [DONATION_INTEGRATION_PLAN.md](DONATION_INTEGRATION_PLAN.md) for full evaluation of alternatives, setup timelines, and implementation roadmap.

Donations support maintenance and long-term quality, but donation prompts should remain subtle and never block app usage.

## Future Policy

- Future paid features are possible, but not committed.
- Any future monetization change must be announced clearly and documented.
- User trust and privacy requirements are non-negotiable.

## Privacy Requirements for Monetization

- No user health data is sold, shared, or used for marketing.
- No invasive tracking is introduced for monetization.
- All health readings remain stored locally on the user's device.
- No account is required for core app usage.

## Placement Rules for Donation CTAs

Allowed:
- README support section
- Footer support link
- Optional support/about page

Not allowed:
- Blocking startup prompts
- Forced popups in core workflows
- Donation gates for core health tracking

## Compliance Notes

- GitHub Pages supports donation links.
- Donation providers should be configured with transparent creator identity and payout details.
- Local legal/tax handling is creator-managed and should be reviewed as needed.

## Summary

PixieTrack stays quality-first and privacy-first: fully free, donation-supported, and user-controlled.
